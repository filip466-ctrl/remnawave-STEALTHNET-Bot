## Подготовка

Для полного тестирования понадобятся:

1. **Аккаунт админа** — вход через `/admin`
2. **Аккаунт клиента №1** (отправитель подарков) — вход через `/cabinet`
3. **Аккаунт клиента №2** (получатель подарков) — другой email/телефон
4. **Telegram** — бот `@vpn_stealthnet_bot` (оба аккаунта привязаны к Telegram)
5. **Баланс** — на аккаунте №1 должно хватить на покупку тарифа

## Фаза 1 — Баги и логика

### ✅ Задача 13: Фильтрация неактивных подписок на дашборде

**Шаги:**
1. Зайти в **Кабинет клиента** (`/cabinet`)
2. На главной странице (дашборд) посмотреть список подписок

**Ожидаемый результат:**
- Подписки со статусом "подарено" НЕ отображаются на главном дашборде ✓
- Видны только активные собственные подписки ✓

---

### ✅ Задача 14: Фикс страницы подписки (UUID)

**Шаги:**
1. Зайти в **Кабинет → Подписка** (кнопка "VPN" или "Подписка")
2. Если у клиента есть дополнительные подписки — каждая должна иметь свою ссылку

**Ожидаемый результат:**
- Страница подписки загружается с правильным UUID ✓
- У каждой подписки своя уникальная ссылка (не одинаковая для всех) ✓

---

### ✅ Задача 15: "Активировать себе"

**Шаги:**
1. Зайти в **Кабинет клиента №1 → Подарки**
2. Купить дополнительную подписку (нажать "Купить подписку", выбрать тариф)
3. Найти купленную подписку в списке
4. Нажать кнопку **"Активировать себе"**

**Ожидаемый результат:**
- Подписка переходит в статус "Активирована" ✓
- В истории появляется запись "Активировал себе" ✓
- Подписка появляется на дашборде ✓

**Проверка через бота:**
1. Открыть бота → Подарки → выбрать подписку
2. Нажать "Активировать себе"
3. Бот должен прислать ссылку на подписку ✓

---

### ✅ Задача 16: Подарок не исчезает у отправителя

**Шаги:**
1. Клиент №1: купить подписку → создать подарочный код
2. Клиент №2: активировать этот код
3. Клиент №1: зайти в **Подарки** и проверить

**Ожидаемый результат:**
- У клиента №1 подписка по-прежнему видна в списке ✓
- Статус изменился на "Подарено" (с информацией кому) ✓
- Подписка НЕ пропала из списка ✓

---

### ✅ Задача 17: Удаление подписок

**Шаги:**
1. Зайти в **Кабинет → Подарки**
2. Найти подписку без активного подарочного кода
3. Нажать кнопку удаления (иконка корзины)
4. Подтвердить удаление

**Ожидаемый результат:**
- Подписка удалена из списка ✓
- В истории появляется запись об удалении ✓

**Через бота:**
1. Подарки → выбрать подписку → "🗑 Удалить"
2. Подтвердить → подписка удалена ✓

**⚠️ Нельзя удалить:**
- Подписку с активным (неиспользованным) подарочным кодом

---

### ✅ Задача 18: Подарок = всегда дополнительная подписка

**Как проверить:**
- При любой операции с подарками (покупка, активация, получение) — подписка НИКОГДА не попадает в основную. Всегда в "Дополнительные подписки"
- Проверить: **Админка → Клиенты** — у клиента не должна меняться основная подписка из-за подарков

---

### ✅ Задача 19: Защита от дубликатов

**Шаги:**
1. Клиент №2 уже имеет подписку с тарифом X (из предыдущего теста)
2. Клиент №1 создает ещё один подарочный код с тем же тарифом X
3. Клиент №2 пытается активировать этот код

**Ожидаемый результат:**
- Ошибка: "У вас уже есть подписка с таким тарифом" ✓
- Код НЕ активируется ✓

---

### ✅ Задача 20: Нельзя активировать свой код

**Шаги:**
1. Клиент №1: купить подписку → создать подарочный код
2. Клиент №1: попробовать ввести СВОЙ код (через "Ввести код" или бота)

**Ожидаемый результат:**
- Ошибка: "Нельзя активировать собственный код" ✓
- Код остается активным ✓

---

### ✅ Задача 21: Автоэкспирация кодов

**Как проверить:**
- Подарочные коды имеют срок действия (по умолчанию 72 часа)
- Если код не активирован вовремя — при попытке активации будет ошибка "Код истёк"
- В **Админка → Настройки → Подарки** можно изменить `Срок действия кода (часы)`

**Примечание:** Для тестирования можно временно поставить маленький срок (например, 1 час), создать код, подождать, и попробовать активировать.

---

### ✅ Задача 22: Rate Limiting

**Как проверить:**
1. Открыть страницу `/gift/XXXX-XXXX-XXXX` с несуществующим кодом
2. Быстро обновить 5+ раз подряд (F5)

**Ожидаемый результат:**
- Первые 5 запросов проходят ✓
- После 5-го — ошибка "Слишком много попыток. Подождите минуту." ✓
- Через минуту снова работает ✓

## Фаза 2 — Админка

## Фаза 3 — Подарочный flow

### ✅ Задача 29: Красивая страница `/gift/:code`

**Шаги:**
1. Клиент №1: создать подарочный код (из кабинета или бота)
2. Скопировать ссылку на подарок
3. Открыть ссылку в **браузере** (не залогиненный)

**Ожидаемый результат:**
- Красивая glassmorphism-страница ✓
- Информация о подарке: тариф, от кого, персональное сообщение (если было) ✓
- Две кнопки: "У меня есть аккаунт" и "Создать аккаунт" ✓
- Если код уже использован — показывается соответствующее сообщение ✓
- Если код истёк — сообщение об истечении ✓

**Проверить на мобилке:**
- Открыть ту же ссылку с телефона → адаптивный дизайн ✓

---

### ✅ Задача 30: Активация на существующий аккаунт

**Шаги:**
1. Клиент №1: создать подарочный код
2. Открыть ссылку на подарок в браузере
3. Нажать **"У меня есть аккаунт"**
4. Авторизоваться как клиент №2
5. Подарок должен активироваться автоматически

**Ожидаемый результат:**
- Редирект на `/cabinet` ✓
- На дашборде появляется баннер "Подарок активирован!" ✓
- Подписка появляется в "Дополнительные подписки" ✓
- В истории клиента №2 — запись "Подарок получен" ✓
- У клиента №1 статус подписки меняется на "Подарено" ✓

---

### ✅ Задача 31: Создание нового аккаунта + реферал

**Шаги:**
1. Клиент №1: создать подарочный код
2. Открыть ссылку на подарок в **новом/приватном браузере**
3. Нажать **"Создать аккаунт"**
4. Зарегистрироваться (новый email)
5. После регистрации подарок активируется автоматически

**Ожидаемый результат:**
- Регистрация проходит ✓
- Подарок активируется автоматически ✓
- Клиент №1 получает реферальный бонус (если `giftReferralEnabled = true` в настройках) ✓
- Новый клиент привязан как реферал клиента №1 ✓

**Проверка реферала:**
- **Кабинет клиента №1 → Рефералы** — новый клиент в списке ✓

**Шаги:**
1. Клиент №1: создать подарочный код (получить код формата XXXX-XXXX-XXXX)

### ✅ Задача 35: Push-уведомления

**Что проверяется автоматически при действиях выше:**

| Событие | Кто получает уведомление |
|---------|------------------------|
| Подарок активирован | **Создатель** получает сообщение в бота: "Ваш подарок для [тариф] был активирован!" |
| Код истекает (за N дней) | **Создатель** получает напоминание |

**Как проверить push:**
- При тесте задачи 30/31/32 — клиент №1 должен получить уведомление в боте ✓

---

### ✅ Задача 36: Cron уведомления об истечении

**Как проверить:**
- Настройка в **Админка → Настройки → Подарки** — "Уведомления об истечении (дни)"
- Cron запускается автоматически и проверяет коды, срок которых истекает через N дней
- Проверить через логи: `docker compose logs -f api | grep expir`

**Практический тест (для терпеливых):**
1. Поставить срок кода = 2 часа, уведомление = за 1 день (не сработает — слишком быстро)
2. Или просто убедиться, что cron запущен: в логах API при старте видно "Gift expiry scheduler started" ✓

## ⚠️ Известные нюансы

1. **Rate limiter** — захардкожен на 5 req/min. Изменение настройки в админке не действует мгновенно (только после перезапуска API контейнера)
2. **Нет тестов** — проект не имеет автоматических тестов, всё проверяется вручную
3. **Pre-existing TS ошибки** — в backend/bot есть 5 ошибок типов (`maxmind`, `undici`, `socks-proxy-agent`) — это НЕ наши баги, они были до Gift System v2
4. **Prisma upgrade** — Prisma предлагает обновиться с 6.19 до 7.7.0 — мажорное обновление, не обязательно сейчас
